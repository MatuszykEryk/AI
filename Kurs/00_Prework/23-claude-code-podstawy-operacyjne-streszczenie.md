### Claude Code — podstawy operacyjne
Claude Code to osobisty agent programowania z modelami Opus lub Sonnet kontrolowany przez harness — warstwę narzędzi, bezpieczeństwa i kontroli wywołań. Harness integruje modele z systemem plików i zasobami zewnętrznymi; to on sprawił, że Claude Code wykorzystuje potencjał lepiej niż wiele alternatyw z tym samym dostępem do modeli: właściwe tool use, terminal, korekta działań, przeszukiwanie plików i aktualna wiedza o projekcie. Jeśli różnica między modelem, agentem a harnessem jest niejasna — wróć do wcześniejszej lekcji.

### Jeśli korzystasz z innego narzędzia
Nie każde narzędzie agentowe działa w terminalu jak Claude Code, ale oceniaj je przez ten sam zestaw pytań; interfejs bywa inny, sprawczość, koszt pracy i bezpieczeństwo zostają po twojej stronie. Uprawnienia: sprawdź, kiedy pyta o zgodę na edycję plików, uruchamianie komend, czytanie danych spoza katalogu projektu, korzystanie z sieci i podłączanie integracji zewnętrznych; tryb bez pytań traktuj jako kontrolowany eksperyment, nie domyślny sposób pracy na ważnym repozytorium — gdy konfiguracja Agent + Model wychodzi poza obszar roboczy, rozważ inne środowisko. Komendy operacyjne (pomoc, wybór modelu, sprawdzenie kontekstu, podgląd zmian, czyszczenie sesji, kompaktowanie rozmowy, kontrola kosztów) to podstawowe metody kontroli, nie ozdobniki dla power-userów. Pamięć projektu: plik konfiguracyjny jak `CLAUDE.md`, reguły projektu lub stałe instrukcje dodawane do każdej sesji; szukaj wsparcia `AGENTS.md` i `.agents/skills`. Alternatywy na dziś: Codex z modelami GPT-5.* oraz OpenCode z Kimi K2.6 lub GLM-5.1 (zwykle ekonomiczniej) — bez benchmarku łatwo porównywać narzędzia po ekranach startowych i marketingu, a to nieprecyzyjna metoda. Sprawdź polityki prywatności: czy i kiedy kod trafia do treningu modeli oraz limity subskrypcji — stała kwota vs zużycie tokenów przez API. Antywzorzec: agent bez kontroli budżetu bywa bardzo przekonujący i bardzo drogi.
